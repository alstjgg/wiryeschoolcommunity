## 1. 기관 개요

### 1.1 위례인생학교란?

위례인생학교는 성인을 대상으로 하는 평생교육 프로그램이다. 다양한 분야의 강좌를 제공하며, 지역 주민들의 평생 학습을 지원한다.

### 1.2 운영 주체

- **운영**: 위례인생학교 사무처 (관리자 2~4명)
- **플랫폼**: 배움숲 (수강신청 포털)
- **홍보**: 네이버 카페

---

## 2. 학기 구조

### 2.1 학기제

연 4학기 운영. 각 학기는 3개월, 강좌당 최대 12회차.

| 학기 | 시작월 | 기간 |
|:---:|:-----:|:----:|
| 1학기 | 1월 | 1월 ~ 3월 |
| 2학기 | 4월 | 4월 ~ 6월 |
| 3학기 | 7월 | 7월 ~ 9월 |
| 4학기 | 10월 | 10월 ~ 12월 |

### 2.2 학기 시작일

- **원칙**: 각 월의 첫 주
- **예외**: 강사 사정에 따라 최대 1주 지연 가능
- **참고**: 모든 수업은 요일이 다르지만 첫 주에 개강

### 2.3 주요 업무 일정

| 시점 | 업무 |
|-----|------|
| 학기 시작 2달 전 | 강사에게 강의 계획서 요청 |
| 학기 시작 1달 전 | 강의 계획서 검토, 배움숲/카페 홍보 게시 |
| 전월 말일까지 | 수강신청 접수 마감 |
| 개강 첫 주 전 토요일 | 강사에게 수강자 목록(출석부) 전달 |
| 개강 1주 후 | 환불 마감 |
| 학기 종료 후 | 출석 체크(OCR), 운영 보고서 작성 |

## 3. 회원 등급 체계

### 3.1 등급 구조

3단계 등급 구조. 승급/강등이 학기 흐름과 입금 상태에 따라 전환된다.

```
[비회원] ──가입비──→ [회원] ──수강료 입금──→ [준회원] ──종강──→ [회원]
                       │
                       └──정회원비──→ [정회원] ──만료학기 경과──→ [회원]
```

| 등급 | 승급 조건 | 수강 신청 | 수강료 | 유효기간 | 만료 시 |
|------|---------|:--------:|:------:|---------|--------|
| 회원 | 가입비 납부 | 가능 | 필요 | 영속 | — |
| 준회원 | 수강료 입금 완료 (1과목 이상) | 등록 완료 | 납부 완료 | 해당 학기 종강까지 | 회원으로 강등 |
| 정회원 | 정회원비 납부 | 가능 | **면제** | ~후년 1학기 | 회원으로 강등 |

비회원은 시스템 외부 상태이며, 가입비 납부 완료 시 회원으로 등록된다.

### 3.2 등급 전환 시점

| 시점 | 전환 내용 |
|------|----------|
| 가입비 납부 완료 | 비회원 → 회원 |
| 수강료 입금 확인 (1과목 이상) | 회원 → 준회원 |
| 정회원비 납부 | 회원 → 정회원 |
| 학기 종강 | 준회원 → 회원 |
| 만료학기 경과 (매 1학기 일괄 체크) | 정회원 → 회원 |

### 3.3 특수 케이스

- **강사**: 자동으로 정회원 자격 부여. 시작/만료 기준은 추후 확정.
- **사무처 직원**: 입사 시 정회원으로 승격. 퇴사 시 해당 학기까지 유지, 다음 학기에 만료.

### 3.4 정회원 수강 등록 처리

정회원은 수강료가 면제이므로, 수강 신청만으로 등록이 완료된다.

**처리 흐름**:
1. 수강신청 접수 확인
2. 회원관리에서 등급 확인
3. 정회원인 경우: 입금현황 = "면제", 등록상태 = "정상등록" 자동 처리
4. 회원/준회원인 경우: 입금 내역 확인 후 등록 처리

## 4. 비용 구조

### 4.1 비용 항목

| 항목 | 기본 금액 | 비고 |
|------|:--------:|------|
| 가입비 | 1만원 | 신규 회원 가입 시. 변동 가능 |
| 수강료 | 2만원 | 전 과목 동일. 변동 가능 |
| 정회원비 | 12만원 | 매년 초(약 2월) 결정. 변동 |

모든 비용은 변동 가능하다. 특히 정회원비는 매년 초에 결정된다.

### 4.2 입금 패턴

입금자가 은행 적요란에 자유롭게 입력하므로, 형식이 통일되어 있지 않다.

| 케이스 | 설명 | 금액 |
|-------|------|:----:|
| 기본 | 단일 과목 수강료 | 2만원 |
| 신규 (별도) | 가입비 + 수강료 각각 입금 | 1만원 + 2만원 |
| 신규 (합산) | 가입비 + 수강료 합산 입금 | 3만원 |
| 다과목 (별도) | 과목별 각각 입금 | 2만원 × N |
| 다과목 (합산) | 여러 과목 합산 입금 | 2만원 × N |
| 정회원비 | 정회원 가입 | 12만원 |
| 중복 | 동일 건 중복 입금 | 환불 처리 필요 |
| 대리 입금 | 가족 등이 대신 입금 | 적요와 의뢰인 불일치 |

### 4.3 환불

- **개강 전**: 전액 환불 가능
- **개강 1주 후**: 환불 불가
- **처리**: 별도 민원 담당자가 처리

## 5. 사용자 식별

### 5.1 이름ID

동명이인 구분을 위해 `이름+핸드폰번호뒷자리`를 사용자 식별자로 사용한다.

- **형식**: `이름` + `핸드폰번호 뒤 4자리` (예: `박민서6804`)
- **용도**: 회원관리, 수강생, 출석부 등 모든 테이블의 기본 키

## 6. 수강 등록

### 6.1 등록 상태

| 상태 | 설명 |
|:---:|------|
| ✅ 정상 | 입금 확인 완료 |
| 🔶 확인필요 | 금액 기반 추정 또는 LLM 판단 (수동 확인 필요) |
| ⚠️ 이름불일치 | 입금자명과 신청자명 불일치 (대리 입금 추정) |
| ❌ 미입금 | 입금 내역 없음 |
| 🔄 중복입금 | 동일 건 중복 입금 |
| 💎 면제 | 정회원 — 수강료 면제 |

### 6.2 입금 대조

은행 입금내역과 수강 신청 목록을 비교하여 입금 상태를 판정하는 핵심 업무이다.

**처리 방식**: 코드 기반 규칙 매칭(80~90%) + LLM 기반 예외 처리(10~20%)

1. 정회원 선처리: 입금현황 = "면제", 등록상태 = "정상등록" 자동 설정
2. 코드 매칭: 의뢰인 이름 → 수강생 이름, 비고 컬럼 → 강좌명, 금액 분류
3. LLM 처리: 비정형 적요 텍스트 해석 (매칭 실패/애매한 건만)
4. 결과를 관리자에게 제시 → 확인 후 시트에 반영
5. 입금 확인된 "회원"은 "준회원"으로 등급 승급

## 7. 강의 관리

### 7.1 강의 계획서

강사가 매 학기 강의 계획서를 제출한다.

**포함 내용**: 강좌명, 강사 정보, 강의 개요, 주차별 계획, 준비물 등

**파일 형식**: PDF만 지원 (HWP 미지원)

**AI 처리**:
- 1순위: 오탈자 수정, 말투 수정, 배움숲 업로드용 카테고리별 복붙 멘트 생성
- 후순위: 내용 보강, 용어 보완, 네이버 카페 멘트 생성

**주의사항**: 과목/강사는 학기마다 변동 가능. 계획서를 통해 해당 학기 과목 정보를 확인한다.

### 7.2 출석 관리

**출석부 구조**: 이름ID, 1회차~12회차, 출석률

- 강좌당 최대 12회차로 구성 (미만 회차도 12컬럼 일괄 생성)
- 출석률 = 출석수 / 회차수 × 100
- 등록상태 "정상등록"인 수강자만 출석부에 포함
- 프린트를 위한 PDF 변환 제공

**출석 체크(OCR)**: 실물 출석부에 체크/사인으로 회차마다 출석 확인 후, 종강 이후 강좌별 1회씩 출석부 사진을 촬영하여 AI(Claude Vision)로 디지털화한다.

### 7.3 휴강/보강

- 강사가 직접 처리
- 처리 후 사무처에 알림

## 8. 데이터 구조

> 이 구조는 개발 편의를 위한 제안(안)이며, 운영 상황에 따라 변경 가능하다.

### 8.1 테이블 구성

| 테이블 | 위치 | 성격 | 용도 |
|-------|------|------|------|
| 회원관리 | 루트 | 영속 | 전체 회원 마스터 (등급, active, 누적 통계) |
| 수강기록 | 루트 | 영속 | 전체 수강 이력 (학기×과목×출석률) |
| 수강생 | 학기 폴더 | 학기별 | 해당 학기 수강 신청 + 입금 관리 |
| 출석부 | 학기 폴더 | 학기별 | 강좌별 출석 체크 |

**테이블 관계**:
```
[회원관리] ←──집계── [수강기록]       ← 둘 다 루트, 영속
     │                    ▲
     │ 이름ID             │ 학기 종료 시 추가
     ▼                    │
[수강생]              [출석부]          ← 둘 다 학기 폴더
```

### 8.2 Google Drive 폴더 구조

```
위례인생학교 자료실/
├── 회원관리.xlsx                    ← 전체 회원 마스터 (영속)
├── 수강기록.xlsx                    ← 전체 수강 이력 (영속)
└── 학사운영(연도별)/
    └── {연도}/
        └── {YYYY}_{N}학기/
            ├── 수강생.xlsx           ← 해당 학기 수강 신청 + 입금
            ├── 출석부/
            │   └── {강좌명}_출석부.xlsx
            ├── 입금내역/
            │   └── {YYYY}_{MM}_{DD}_입금내역.xlsx
            └── 강의계획서/
                ├── 초안/
                └── 검토완료/
```

### 8.3 회원관리 테이블 (루트, 영속)

전체 회원(과거+현재)을 관리하는 마스터 테이블.

| 컬럼 | 설명 |
|------|------|
| 이름ID | PK. 이름+뒷자리 (예: 박민서6804) |
| 이름 | |
| 성별 | |
| 전화번호 | |
| 주소 | |
| 나이 | |
| 등급 | 회원 / 준회원 / 정회원 |
| 가입날짜 | |
| 시작학기 | 가입한 학기 |
| 만료학기 | 정회원: 후년 1학기, 회원/준회원: 없음 |
| 수강 count | 누적 수강 횟수 (수강기록에서 집계) |
| 출석률 (누적) | 전체 평균 출석률 (수강기록에서 집계) |
| 마지막수강학기 | (수강기록에서 집계) |
| active | Y/N |

### 8.4 수강기록 테이블 (루트, 영속)

전체 수강 이력. 학기 종료 시 출석부 데이터를 바탕으로 행을 추가한다.

| 컬럼 | 설명 |
|------|------|
| 이름ID | 회원관리와 연결 |
| 학기 | 2026_1학기 등 |
| 과목명 | |
| 출석률 | 해당 강좌의 출석률 |

### 8.5 수강생 테이블 (학기 폴더, 학기별)

해당 학기의 수강 신청자와 입금 상태. 학기마다 새로 생성.

| 컬럼 | 설명 | 입력 주체 |
|------|------|----------|
| 이름ID | PK(+과목명). 회원관리와 연결 | 초기 입력 |
| 과목명 | | 초기 입력 |
| 입금시간 | 매칭된 입금 건의 거래일시 | AI 자동 |
| 입금자명(적요) | 매칭된 입금 건의 은행 적요 (원본) | AI 자동 |
| 비고 | | |
| 입금현황 | ✅정상 / 🔶확인필요 / ❌미입금 / 💎면제 | AI 자동 |
| 등록상태 | 정상등록 / 대기 / 취소 | 관리자 확인 |

### 8.6 출석부 (학기 폴더, 강좌별 파일)

| 컬럼 | 설명 |
|------|------|
| ID | 이름+뒷자리 |
| 1회차 ~ 12회차 | 출석/결석/지각 등 |
| 출석률 | 출석수 / 회차수 × 100 |

## 9. 외부 시스템

### 9.1 배움숲

수강신청 포털.

**주요 기능**: 강좌 목록 게시, 수강신청 접수, 신청 상태 관리

**관리자 작업**: 강좌 정보 입력, 신청자 입금 확인 후 상태 변경

**API**: 미제공. 데이터는 수동으로 Google Drive에 업로드하여 관리.

### 9.2 네이버 카페

홍보 채널. 학기별 강좌 안내, 수강신청 안내, 공지사항 게시.

## 10. 용어 정리

| 용어 | 설명 |
|-----|------|
| 이름ID | 사용자 식별자. `이름+핸드폰뒤4자리` (예: 박민서6804) |
| 회원 | 가입비 납부 완료. 수강 신청 자격. 영속 |
| 준회원 | 수강료 입금 완료 회원. 해당 학기 종강까지 유효 |
| 정회원 | 정회원비 납부 회원. 수강료 면제. 만료학기까지 유효 |
| 배움숲 | 수강신청 포털 사이트 |
| 가입비 | 신규 회원 가입 시 납부 비용 (기본 1만원, 변동 가능) |
| 수강료 | 강좌 수강을 위해 납부하는 비용 (기본 2만원, 변동 가능) |
| 정회원비 | 정회원 가입 비용 (기본 12만원, 매년 초 결정, 변동) |
| 개강 | 해당 학기 첫 수업 시작 |
| 종강 | 해당 학기 마지막 수업 종료 |
| 입금 대조 | 은행 입금내역과 수강 신청 목록을 비교하여 입금 상태를 판정하는 업무 |
| 출석 체크 | 종강 후 실물 출석부를 사진 촬영하여 AI(OCR)로 디지털화하는 업무 |
